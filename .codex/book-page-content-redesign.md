# Book Page Content Redesign

## 状态

- 已确认：2026-07-14
- 范围：展开魔法书的内容布局、视觉贴合和相关提示词交互
- 不包含：重新生成书本底图、让文字随书页弯曲、改变 Electron 窗口尺寸

## 目标

让管理界面真正属于左右两张书页，而不是把普通双栏网页覆盖在书本图片上。文字必须保持平直、清晰和可点击；书页贴合感由独立页面坐标、透明遮罩、墨迹状态和留白完成。

## 页面结构

- 左页和右页使用独立内容容器，不再共享普通双栏网格。
- 左页可书写区域约为 `240-260px`，用于项目目录。
- 书脊保留约 `110-130px` 禁入区，任何文字、控件、悬停底纹不得跨越。
- 右页可书写区域约为 `330-350px`，用于搜索、筛选和提示词目录。
- 最终坐标必须根据 `panel-book-ui-v3b-alpha.png` 的实际书页边界校准，近似值不能替代视觉验收。
- 左右页分别使用与底图匹配的透明页面遮罩。遮罩只裁切底纹、悬停态和滚动内容，不扭曲文字。

## 左页：项目目录

- 左页整体就是项目目录，不显示独立侧栏边框或整块矩形底色。
- 主标题为“项目目录”，`Prompt Archive` 作为小型花体副标题。
- 只保留一个“提示词库”入口，不再同时展示“收件箱”和“全局库”。
- 项目名称、数量和状态按古籍目录排列，使用浅墨分隔或点线，不使用卡片。
- 当前浏览项目使用深墨下划线；用户设置的“当前项目”使用独立金色书签，两种状态不可混淆。
- “当前项目”只作为持久化工作上下文标记，不影响双击捕获，也不会随浏览项目自动变化。
- 置顶项目使用小型金色星标并优先排列。
- 重命名、删除、置顶等低频操作仅在悬停或更多菜单中显示。

### 新建项目

- “项目目录”标题旁显示一个小型 `＋`。
- 点击后，输入行从 `＋` 右侧横向展开，保持在标题行内。
- 输入项目名后按 `Enter` 或点击确认创建。
- 按 `Esc` 或点击输入区外取消。
- 输入框使用标题行剩余宽度；长名称在输入框内部滚动，不得撑出左页安全区。
- 创建成功后自动选中新项目，但不自动设为“当前项目”。

## 右页：手稿目录

- 提示词以连续手稿目录条目展示，不使用独立卡片或整块矩形背景。
- 每条默认展示标题、一行正文摘要、关键关键词、评分入口、置顶状态和复制入口。
- 条目之间只使用浅墨分隔线。
- 项目和阶段显示为文字元信息；修改归属进入更多菜单或完整编辑，不在列表中常驻下拉框。
- 编辑、删除、移动等低频操作进入更多菜单。
- 悬停态使用轻微墨水晕染，并受右页遮罩约束。
- 搜索、待归纳筛选、阶段和排序表现为页眉题头与墨线标签，不使用现代卡片外壳。
- 目录每页固定最多 3 条，页底使用小型左右翻页符号和页码；提示词列表不再纵向滚动。
- 翻页动效只作用于目录书写区，不移动搜索、筛选、书页底图或书框。
- 更多菜单按右页剩余空间决定展开方向，禁止使用“最后一条固定向上”之类的位置猜测。

## 唯一提示词库与待归纳状态

- 产品只有一个提示词库。
- 双击捕获后的提示词直接进入提示词库；`projectId` 为空时标记为“待归纳”。
- 左页不再提供独立收件箱入口。
- 右页顶部提供“待归纳 N”筛选；存在待归纳内容时显示“记得归纳清理哟~”。该提示是页边批注，不是按钮。
- 开启待归纳筛选时自动切回提示词库；选择任一项目时自动关闭待归纳筛选。
- 提示词加入项目后取消待归纳状态，但仍保留在完整提示词库中。
- 搜索默认覆盖整个提示词库。
- 首次打开显示完整提示词库；后续恢复上次的提示词库或项目视图，但不恢复旧搜索词。

## 评分

- 提示词条目默认只显示“评分”按钮；已有评分时显示“评分 N”。
- 点击按钮后在条目内展开五颗星，不打开大型弹窗。
- 五颗星拥有相同、独立且完整的点击区域，任何一颗都不得被裁切或覆盖。
- 点击某颗星后保存 `1-5` 的评分，更新按钮文字并收起。
- 点击条目外或按 `Esc` 收起，不改变原评分。
- 第五颗星必须能够可靠保存评分 `5`，作为强制回归场景。

## 删除提示词

- 删除提示词是正式能力，用于处理误捕获和失效内容。
- 删除入口位于提示词的更多菜单中。
- 点击删除后显示简短确认，明确提示将永久删除该提示词。
- 只有确认后才从唯一提示词库永久移除；取消不产生任何数据变化。
- 不引入回收站或软删除。
- 删除后，提示词必须从提示词库、项目视图、待归纳筛选和搜索结果中同时消失。

## 技术边界

- 文字和控件继续使用语义化 DOM，保持清晰、可访问和准确点击。
- 不使用 Canvas 或 WebGL 将文字栅格化到书页纹理。
- 不用 CSS 重新绘制书本；透明遮罩从当前书页资产的实际边界派生。
- 页面坐标和书脊禁入区应集中定义，避免左右栏再次独立漂移。
- 替换后的旧侧栏、卡片和收件箱入口样式及测试应删除，不并行保留。

## 验收标准

- 左右内容分别落在对应书页内，任何文字和交互底纹都不跨越书脊。
- 内容不触碰装饰边框、宝石和书页曲线，底部不越出页面。
- 左页视觉上是项目目录，不像侧栏；右页视觉上是手稿目录，不像卡片列表。
- 页面中不存在可辨识的整块矩形面板底色。
- 首次打开只有一个提示词库入口，“待归纳”作为筛选存在。
- 新建项目输入行从标题旁 `＋` 的右侧展开，并保持在左页内。
- 五颗星均可点击，第五颗能保存为 `5`。
- 删除提示词需要确认，确认后从所有视图永久移除。
- 4 条提示词稳定分为两页 `3 + 1`；搜索和筛选回到第 1 页，删除末页唯一条目后自动回退。
- 唯一条目与末条提示词的更多菜单均完整可见，不被书写区遮罩裁掉。
- 左右方向键仅在非交互区域翻页，减少动画设置下不播放翻页动效。
- 搜索、复制、置顶、评分、项目选择和阶段筛选仍可完整使用。
- 自动化测试覆盖页面结构、第五颗星、删除确认和唯一提示词库视图；桌面端需要实际截图和点击验证。
