# Prompt Pagination And Menu Placement

## 状态

- 已确认：2026-07-16
- 范围：右页提示词目录分页、翻页反馈、更多菜单空间判断
- 不包含：修改魔法书底图、单条详情阅读模式、安装包与开机自启

## 问题

当前 `.prompt-list` 是带裁切的滚动容器。唯一一条提示词同时匹配 `:last-child`，菜单被固定向上展开，越过列表顶部后被 `overflow` 裁掉，因此点击菜单按钮后看不到菜单。

长滚动列表也会强化“网页列表覆盖在书本上”的感觉，不符合手稿目录的翻阅体验。

## 已确认方案

- 采用列表分页，不采用一次只展示一条提示词的阅读模式。
- 每页固定显示 3 条提示词。
- 右页底部显示上一页按钮、`当前页 / 总页数`、下一页按钮。
- 翻页按钮使用熟悉的 `‹` 与 `›` 符号，不使用带文字的圆角按钮。
- 支持键盘 `ArrowLeft` 与 `ArrowRight`；输入框、文本域、下拉框、按钮、弹窗或可编辑元素聚焦时不触发。
- 搜索词、项目、待归纳、阶段或排序方式变化后回到第 1 页。
- 删除导致当前页超出总页数时，自动回到最后一个有效页。
- 没有结果时页码显示 `0 / 0`，两侧按钮禁用。
- 只有 1 页时保留稳定的页脚空间，按钮禁用，避免列表高度变化。

## 翻页动效

- 动效时长约 180ms。
- 下一页内容轻微向左退出，新内容从右侧进入；上一页方向相反。
- 同时使用低幅度透明度变化和书页阴影扫过，不执行整张底图翻转。
- 动效不得改变列表、页脚或书页的稳定尺寸。
- `prefers-reduced-motion: reduce` 时关闭位移与阴影动画。

## 菜单定位

- 移除基于 `:last-child` 的固定向上规则。
- 菜单打开并完成渲染后，比较菜单、条目与提示词列表的实际边界。
- 下方空间足够时向下展开；不足且上方空间足够时添加向上展开状态。
- 两侧都不足时限制菜单最大高度并允许菜单自身滚动，不允许越出右页列表边界。
- 切页、搜索、筛选、排序和关闭面板时关闭当前菜单。

## 状态模型

```text
PROMPTS_PER_PAGE = 3
currentPromptPage = 1
promptPageDirection = forward | backward | none
totalPromptPages = ceil(filteredPromptCount / PROMPTS_PER_PAGE)
visiblePrompts = filteredPrompts.slice(pageStart, pageEnd)
```

页码是纯界面状态，不写入提示词数据模型，也不在应用重启后恢复。

## 验收标准

- 只有一条提示词时，更多菜单向下展开且完整可见。
- 每页最多显示 3 条提示词。
- 第 4 条提示词出现在第 2 页。
- 左右按钮、键盘方向键和页码状态一致。
- 搜索、项目、待归纳、阶段和排序变化后进入第 1 页。
- 删除最后一页唯一条目后自动回到上一页。
- 翻页过程中提示词不越出列表和右书页边界。
- 菜单操作、复制、评分、编辑、删除、项目归纳和搜索排序仍可使用。
- 自动化测试覆盖分页计算、状态回退、键盘守卫、菜单方向和 reduced-motion；Electron 中完成实际点击与边界检查。
