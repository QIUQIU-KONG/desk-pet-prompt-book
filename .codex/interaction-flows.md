# Interaction Flows

## 1. 双击桌宠捕获

目标：快速保存剪贴板提示词，不打断当前工作。

主路径：

1. 用户复制提示词。
2. 用户双击桌宠。
3. 系统读取剪贴板文本。
4. 如果为空，显示轻量反馈。
5. 如果正文完全重复，显示“已经收过啦”，不新增。
6. 如果有效，生成标题并保存到提示词库。
7. 新提示词没有项目归属，因此标记为待归纳。
8. 桌宠显示捕获成功反馈。

验收标准：

- 双击不会自动打开小浮窗。
- 空剪贴板不会新增提示词。
- 完全重复内容不会新增提示词。
- 新提示词可通过待归纳筛选和全局搜索找到。

## 2. 单击桌宠打开小浮窗

目标：进入提示词管理和复用界面。

主路径：

1. 用户单击桌宠。
2. 小浮窗打开。
3. 系统加载提示词库中的真实提示词数据。
4. 首次打开显示完整提示词库。
5. 非首次打开恢复上次停留视图。
6. 如果上次停留项目已删除，回到提示词库。

验收标准：

- 单击可以打开或关闭小浮窗。
- 小浮窗不恢复旧搜索词。
- 存在待归纳提示词时，右页显示“待归纳 N”和“记得归纳清理哟~”。

## 3. 浏览视图

目标：用左页项目目录和右页筛选切换浏览范围。

视图：

- 提示词库：展示所有提示词。
- 待归纳筛选：只展示未归入项目的提示词。
- 项目：展示该项目内提示词。

验收标准：

- 左页是完整项目目录，不显示独立侧栏矩形边界。
- 无搜索词时，左页选中项和待归纳筛选共同决定右页列表。
- 开启待归纳筛选时切回提示词库；选择项目时关闭待归纳筛选。
- 选择项目不会改变提示词捕获规则。
- 选中项目和用户设置的“当前项目”使用不同视觉状态。

## 4. 搜索

目标：让搜索成为主要查找方式。

主路径：

1. 用户在搜索框输入关键词。
2. 系统默认搜索整个提示词库。
3. 搜索覆盖标题、正文、备注、关键词。
4. 结果按提示词展示。
5. 用户可切换“仅当前视图”缩小范围。

验收标准：

- 有搜索词时默认全局搜索。
- 没有结果时显示空状态。
- 清空搜索后回到当前提示词库状态或项目视图。

## 5. 点击提示词复制

目标：高频复用提示词。

主路径：

1. 用户点击提示词条目主体。
2. 系统复制提示词正文到剪贴板。
3. 系统更新 `lastUsedAt` 和 `useCount`。
4. 系统显示轻量反馈。

验收标准：

- 条目主体点击不打开详情。
- 复制的是最新正文。
- 删除按钮不在主体点击区域内。

## 6. 快速归纳到项目

目标：从提示词库或待归纳筛选快速整理提示词。

主路径：

1. 用户打开提示词更多菜单。
2. 用户选择“加入项目”或“移动项目”。
3. 用户选择目标项目。
4. 系统更新提示词 `projectId`。
5. 阶段不强制选择；未选阶段显示为“未分阶段”。

验收标准：

- 快速归纳不打开完整编辑弹窗。
- 提示词加入项目后取消待归纳状态，但仍保留在提示词库。
- 从项目移出后重新进入待归纳状态。

## 7. 编辑提示词

目标：完整维护提示词内容和元信息。

主路径：

1. 用户通过提示词更多菜单或编辑按钮打开编辑弹窗。
2. 用户编辑标题、正文、备注、关键词、评分、置顶、项目、阶段。
3. 用户保存。
4. 弹窗关闭，列表保持原浏览上下文。

验收标准：

- 编辑使用弹窗，不挤压列表。
- 保存后列表和搜索结果反映最新内容。
- 阶段选择只展示当前项目的阶段。

## 8. 项目管理

目标：维护具体工作流或复盘空间。

主路径：

1. 用户点击“项目目录”标题旁的 `＋`。
2. 输入行从 `＋` 右侧在标题行内展开。
3. 用户输入项目名并按 `Enter` 或点击确认；按 `Esc` 或点击外部取消。
4. 系统创建项目并复制默认阶段模板。
5. 新项目自动选中，但不会自动设为“当前项目”。
6. 用户可重命名、删除、置顶项目。
7. 用户可显式设置“当前项目”；浏览其他项目不会改变该标记。
8. 项目操作入口通过右键、更多菜单或 hover 显示。
9. 重命名在书内文本框完成；删除在书内确认框完成，`Esc` 取消且不修改数据。

验收标准：

- 新项目自动拥有默认阶段。
- 新建项目输入行不得越出左页安全区。
- 删除项目前需要确认。
- 删除项目不删除提示词本体。
- 删除被设为“当前项目”的项目时，当前项目标记被清空。
- 项目目录默认保持干净，不显示现代侧栏外壳。

## 9. 阶段管理

目标：让项目支持流程复盘。

主路径：

1. 用户进入某项目。
2. 用户通过阶段筛选查看提示词。
3. 用户可管理该项目阶段：新增、重命名、隐藏、排序。
4. 未选阶段的提示词显示为“未分阶段”。
5. 阶段过多时，用户可在阶段区域横向滚动；新增阶段后自动定位到末端。
6. 新增/重命名使用书内文本框，隐藏使用书内确认框。

验收标准：

- 修改某项目阶段不影响其他项目。
- 被隐藏阶段不作为默认筛选展示。
- 阶段筛选不影响全局搜索默认范围。
- `＋阶段` 和排序按钮在横向滚动时保持可见。

## 10. 评分

目标：在不常驻五颗星的情况下快速设置评分。

主路径：

1. 用户点击“评分”或“评分 N”按钮。
2. 条目内展开五颗星。
3. 用户点击某颗星，系统保存 `1-5` 的评分。
4. 评分按钮更新并自动收起。
5. 点击条目外或按 `Esc` 只收起，不修改评分。

验收标准：

- 五颗星拥有相同且互不重叠的点击区域。
- 第五颗星能够可靠保存评分 `5`。
- 评分展开区域不越出右页遮罩或触发条目复制。

## 11. 删除

目标：清理无用提示词和项目。

提示词删除：

1. 用户选择删除提示词。
2. 系统展示永久删除确认。
3. 用户确认后，提示词从提示词库移除。

项目删除：

1. 用户选择删除项目。
2. 系统展示书内确认框。
3. 用户确认后，项目删除。
4. 项目内提示词进入待归纳状态。

验收标准：

- 删除提示词需要确认。
- 取消删除时数据不发生变化。
- 确认删除后，提示词从提示词库、项目、待归纳筛选和搜索结果中同时消失。
- 删除项目需要确认。
- 删除项目不删除提示词本体。
- 管理流程不调用 Electron 不支持的原生 `prompt()` 或 `confirm()`。

## 12. 提示词翻页与更多菜单

目标：在书页安全区内稳定浏览提示词，并保证任何位置的条目菜单都完整可用。

主路径：

1. 系统完成搜索、视图筛选和排序后，每页展示最多 3 条提示词。
2. 用户点击页底上一页/下一页，或在非交互区域按左右方向键翻页。
3. 旧页向翻阅方向轻微淡出，新页从反方向进入；系统启用减少动画时直接切换。
4. 用户打开更多菜单时，系统测量书写区、触发按钮和菜单高度。
5. 菜单优先向空间充足的一侧展开；两侧都不足时限制菜单高度并允许菜单内部滚动。

状态规则：

- 搜索词、搜索范围、项目/提示词库、待归纳、阶段或排序变化后回到第 1 页。
- 编辑和复制不主动改变页码。
- 删除或移动最后一页唯一条目后，自动回到新的最后一页。
- 输入框、下拉框、按钮、可编辑区域或弹窗获得焦点时，左右方向键不触发翻页。

验收标准：

- 4 条提示词按 `3 + 1` 分布，页码显示 `1 / 2` 和 `2 / 2`。
- 空结果显示 `0 / 0`，首尾页按钮禁用状态正确。
- 唯一条目、首条、中间条和末条的更多菜单均完整显示在右页书写区内。
- 搜索期间发生的旧翻页动画不得覆盖最新搜索结果。
- 提示词列表本身不出现纵向滚动条，任何内容不越出右页边界。

## 13. 重复启动与退出桌宠

目标：让无任务栏、无系统标题栏的桌宠拥有明确退出路径，并保护本地提示词文件不被多个应用进程同时写入。

主路径：

1. 用户再次启动应用时，新的进程尝试获取单实例锁。
2. 如果已有实例持有锁，新进程退出，并让已运行的桌宠或小浮窗显示并获得焦点。
3. 用户在桌宠状态或展开面板上右键。
4. 系统打开原生菜单，只显示“退出桌宠”。
5. 用户选择该命令后，应用进入正常退出流程。

验收标准：

- 同一数据目录只有一个应用实例负责读写。
- 第二次启动不会创建第二个桌宠窗口。
- 右键本身不会立即退出，必须选择“退出桌宠”。
- 桌宠和展开面板都能打开同一个原生退出菜单。
