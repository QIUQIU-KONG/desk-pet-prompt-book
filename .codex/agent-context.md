# Agent Context

## 一句话

这是一个桌宠式提示词工作流管理工具：双击桌宠捕获提示词，单击桌宠打开小浮窗，通过搜索、项目、阶段、关键词、评分和置顶管理提示词。

## 已确认方向

- 从 0 开始搭建，不复用已解包应用代码。
- 桌宠负责捕获、入口和轻量反馈。
- 小浮窗是主要管理界面。
- 前端目标形态是独立左右书页：左页项目目录，右页手稿目录式提示词列表，中间避让书脊。
- 桌宠外观方向已确认：漂浮展开古代魔法书，参考 `.codex/ui-visual-direction.md`。
- 搜索是首选查找方式。
- 项目是具体工作流或复盘空间，不是必须维护的分类负担。
- 界面只有一个提示词库入口；未归入项目的提示词以“待归纳”状态筛选。
- 取消当前捕获项目概念。
- 取消多项目引用模型。
- 提示词以单项目归属为主；跨项目复用可通过复制生成副本。
- 阶段采用“默认模板复制到项目内，项目独立调整”。
- 开发不采用临时 MVP 心态；旧代码和废弃逻辑及时删除。

## 核心对象

- `Prompt`：提示词本体。
- `Project`：具体工作流或复盘空间。
- `Stage`：项目内流程阶段。
- `Keyword`：可复用关键词。
- `Pending`：`projectId` 为空时的待归纳状态，不是独立容器或入口。

## 桌宠交互

- 双击：读取剪贴板并捕获提示词。
- 单击：打开或关闭小浮窗。
- 捕获后只轻量反馈，不自动打开小浮窗。

## 小浮窗交互

- 默认恢复上次停留视图；首次打开进入完整提示词库。
- 有搜索词时默认全局搜索。
- 无搜索词时由左侧视图决定展示范围。
- 点击提示词条目复制正文。
- 评分默认折叠；编辑、置顶、项目归纳和永久删除进入手稿条目的更多菜单。
- 项目重命名、阶段新增/重命名使用书内文本对话框；项目删除、阶段隐藏使用书内确认框，不调用 Electron 不支持的原生 `prompt/confirm`。
- 阶段筛选横向滚动，`＋阶段` 与排序按钮固定可见，新阶段创建后自动定位到末端。
- 右页提示词每页固定最多 3 条，页底箭头和非交互区域左右方向键翻页；搜索、范围、视图、待归纳、阶段和排序变化后回到第 1 页。
- 提示词更多菜单必须按真实几何空间选择上下方向，禁止恢复 `.prompt-entry:last-child` 定位规则。

## 不要做

- 不要做成普通剪贴板历史工具。
- 不要恢复“当前捕获项目”概念。
- 不要恢复“同一提示词多项目引用”模型。
- 不要恢复独立收件箱入口、项目或存储容器。
- 不要让小浮窗变成大型后台。
- 不要为了短期交付写无法承接最终方向的临时代码。

## 当前实现状态

- 已建立 Electron 桌宠壳：透明、无边框、置顶、不进任务栏，加载魔法书桌宠预览。
- 已接入工程选定的 `pet-book-body-v5-alpha.png` 透明魔法书资产；浏览器预览保留 CSS 光效，Electron 使用独立 RGBA 光柱与蝴蝶图层，动画仅改变透明度和位移缩放，避免透明窗口方形残影。
- 已建立剪贴板捕获链路：渲染层双击 -> preload 安全桥 -> 主进程读取剪贴板 -> 核心存储模块保存提示词。
- 已实现捕获规则：空文本不保存、完全重复正文不重复保存、标题来自正文第一条非空行、新提示词默认没有项目归属。
- 已建立 JSON 存储，模型包含 `prompts/projects/stages/keywords/viewState`，并已接入提示词编辑、删除、项目归纳、项目置顶/重命名/删除、阶段新增/重命名/隐藏/排序、关键词复用、搜索排序和视图状态恢复。
- 小浮窗内容页重构已完成：唯一提示词库、待归纳筛选、独立左右书页、120px 书脊禁入区、手稿条目、折叠评分、更多菜单和永久删除确认均已接入真实数据。
- 手稿目录已改为每页 3 条的固定分页：支持页底按钮、受保护的左右方向键、删除后页码钳制、约 180ms 方向动效与减少动画降级；更多菜单使用书页几何测量，唯一条目不再因强制向上展开而被裁掉。
- 2026-07-16 已用 Playwright 直连 Electron `39.8.10` 做隔离内存验收：窗口从 `220×220` 切换到 `1024×700`，4 条提示词稳定显示为 `3 + 1`，每条高度 `77px` 且列表高 `231px`；搜索回首页、输入控件方向键保护、普通方向键翻页、删除末页唯一条目回退和首尾菜单几何方向均通过，未写入真实提示词数据。
- 项目浏览状态与“当前项目”独立；项目标题旁支持行内新建，新建只选中项目，不自动设置当前项目。
- 浏览器预览模式不再伪报编辑、复制或捕获成功：编辑会更新预览模型，复制/捕获会调用浏览器剪贴板 API。
- 已验证浏览器 `1280×720` 和 Electron `220×220 / 1024×700` 窗口模式、透明边角、拖动尺寸稳定、评分 5、项目/阶段、搜索、排序、删除及重启持久化。
- 2026-07-15 最终隔离验收已连续两次跑通 Electron `v39.8.10`：应用内复制正文后双击桌宠，真实剪贴板捕获返回重复状态且总数不变，单击/双击分离成立。
- 跨屏拖动从 `devicePixelRatio=2` 的显示器进入 `devicePixelRatio=1.25` 的显示器时，拖动中、停止后和稳定后窗口均保持 `220×220`。
- 隔离数据重启后已再次验证标题/正文编辑、备注、关键词、五星、置顶、项目、阶段、当前项目、使用次数和删除持久化。
- 阶段 8 稳定性已完成：写操作串行化，主文件与 `.bak` 双快照，损坏文件隔离恢复，孤儿关系规范化，256 KiB 正文、2,000 条搜索和完整状态重建均有回归覆盖。
- Electron 面板模式切换已幂等化，重复打开/关闭不会覆盖桌宠原始位置和 `220×220` 尺寸。
- 当前可运行命令：`corepack pnpm start` 启动桌宠，`corepack pnpm test` 跑测试，`corepack pnpm run readiness` 执行严格仓库检查。

## 开发执行基线（2026-07-15）

- 当前机器热缓存下：冻结安装约 `0.5s`、语法检查约 `1.2s`、90 项测试约 `1.5s`、readiness 约 `0.7s`、依赖审计约 `3.3s`；完整本地质量门通常低于 `10s`。

- 开发中运行相关测试，交付前只运行一次完整质量门并在最终推送后检查一次 CI。
