# 隐私说明

## 适用范围

本说明适用于从本仓库源码运行的 Desk Pet Prompt Book。当前项目处于源码开发阶段 Alpha，没有账户、云服务、遥测或分析系统。

## 剪贴板（Clipboard）访问

Electron 桌宠只在用户执行明确捕获手势后读取剪贴板文本：当前交互是双击桌宠。单击桌宠只展开管理面板，不读取剪贴板。

当用户在提示词列表中执行复制操作时，应用会把对应提示词正文写入系统剪贴板，并更新本地的最近使用时间和使用次数。系统剪贴板的后续生命周期由操作系统和其他本地软件控制。

## 本地存储

应用把以下内容保存在 Electron `userData` 目录下的 `data/prompts.json`，并在同一目录维护最新成功快照 `prompts.json.bak`：

- 提示词标题、正文、备注、评分、置顶和使用记录；
- 项目、阶段与关键词；
- 最近打开的视图、当前项目标记、筛选和排序状态。

Windows 的常见数据位置是 `%APPDATA%\desk-pet-prompt-book\data\prompts.json`。实际位置由 Electron 和当前系统账户决定。

The local JSON data is **not encrypted at rest**. 能够访问当前系统账户文件的人或程序可能读取提示词内容。不要把此应用当作密码库或机密信息保险库。

## 网络行为

当前运行时代码没有设计提示词库远程上传、遥测、分析、账户登录或云同步。桌面应用从本地文件加载渲染器与视觉资源。

安装依赖、访问 GitHub、运行依赖审计以及使用浏览器预览服务会产生开发工具层面的网络或本地 HTTP 流量；这些行为不等于应用上传提示词库。

## 浏览器预览差异

浏览器预览用于开发和布局检查。浏览器能否读取或写入剪贴板取决于浏览器授权、安全上下文和页面焦点。拒绝授权时，预览中的捕获或复制可能不可用；Electron 桌面模式通过受限 preload API 访问主进程剪贴板能力。

## 日志

Electron 主进程只在未捕获异常、未处理 Promise 拒绝、页面加载失败、渲染进程异常退出或提示词存储恢复时，向 `userData/startup-log.txt` 追加诊断信息。存储恢复日志只记录恢复类型、原因和本地文件名，不记录提示词正文。日志没有设计远程上传，但错误文本可能包含本地路径或运行时上下文，分享前应先检查并脱敏。

## 删除与备份

- 删除单条提示词：在应用内执行删除并确认，记录会从本地提示词库永久移除。
- 删除全部提示词数据：先退出应用，再删除 `userData/data/prompts.json`、`prompts.json.bak` 和同目录下的 `prompts.json*.corrupt-*` 恢复文件。
- 删除全部应用状态：先退出应用，再删除该应用的整个 `userData` 目录；这也会移除缓存和本地诊断日志。
- 备份：应用会维护最近一次成功写入的 `prompts.json.bak`。手动备份应在应用退出后复制 `prompts.json` 和 `.bak`；恢复前应保留原文件，并确认备份来自可信来源。

删除本地数据不会清理操作系统剪贴板历史、文件系统备份、同步软件副本或其他程序已经复制的内容。

## 隐私相关变更

任何新增网络请求、遥测、账户、云同步、加密、导入导出或剪贴板后台监听功能，都必须在实现前经过单独的隐私评审，并同步更新本说明与 [`docs/data-and-privacy.md`](docs/data-and-privacy.md)。
